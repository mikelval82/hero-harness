Sesion de alineacion profunda antes de crear o modificar documentacion. Te interrogo exhaustivamente para asegurar que el contenido que vamos a construir encaja con tus objetivos, el contexto del proyecto y los estandares existentes.

Tema o documento a explorar: $ARGUMENTS

## Proceso

### 1. Contextualizar

Explora la estructura de documentacion del proyecto:
- Busca README, indices o documentos principales con Glob (`**/README.md`, `**/_index.md`, `**/INDEX.md`).
- Identifica las convenciones de documentacion existentes (formato, estructura, estilo).
- Revisa que documentacion ya existe sobre el tema para evitar duplicacion.

### 2. Interrogar — una pregunta a la vez

Haz las preguntas en este orden, esperando respuesta antes de continuar:

**Nivel de entrada:**
- Que sabes ya sobre este tema? Has trabajado con ello antes?
- Hay algun concepto previo que no quedo claro y necesita repaso?

**Objetivos:**
- Que quieres que el lector sea capaz de hacer/entender al terminar?
- Te interesa mas la vision conceptual, los detalles tecnicos, o la guia practica?
- Hay un caso de uso concreto que quieras cubrir?

**Alcance:**
- Hasta que nivel de profundidad quieres llegar?
- Hay subtemas que quieras excluir explicitamente?
- Quieres comparativas con alternativas o solo el enfoque elegido?

**Conexiones:**
- Que conexiones con otros documentos del proyecto te interesan?
- Hay documentacion externa que deba referenciarse?

### 3. Desafiar

- Si el alcance parece demasiado amplio para un solo documento: propon una particion.
- Si detectas que falta contexto previo: sugiere documentar eso primero.
- Si la documentacion ya existe parcialmente: senala solapamientos.

### 4. Cristalizar

Cuando todas las ramas esten resueltas, presenta un **brief del documento**:

- Titulo y ubicacion propuesta
- Nivel de profundidad acordado
- Secciones a cubrir (con orden)
- Temas excluidos
- Conexiones con documentacion existente
- Tipo de contenido: narrativo / tecnico / guia practica / referencia

El usuario confirma el brief y entonces se procede a crear el contenido.
